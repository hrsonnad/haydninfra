Workflow Kit
Reusable playbooks, templates, and agent prompts for Linear-centered multi-agent delivery.

Quick Setup

Add the workflow kit to any project in three steps.

  1. Clone the shared repo (one time only):
    git clone https://github.com/hrsonnad/workflow-kit.git ~/workflow-kit
  2. From any project root, sync the docs in:
    ~/workflow-kit/bin/sync-into-project.sh .
  3. Add a Makefile target for easy re-syncing:
    ~/workflow-kit/bin/add-sync-workflow-target.sh .

Daily Commands

Sync workflow docs
make sync-workflow
Check sync status
make workflow-status

What Gets Synced

Running make sync-workflow copies these into docs/agentic-workflow/ in your project repo.

playbooks/
Brain-dump intake, landing-page workflow, and the linear command center checklist.
templates/
Linear epic/story templates, status comments, and per-agent prompt packs.
metrics/
Pilot measurement templates for cycle time, rework, and artifact completeness.

Updating the Kit

All changes live in ~/workflow-kit. Edit files there, commit, and push. Then re-sync each project repo.

  1. Edit playbooks, templates, or prompts in ~/workflow-kit
  2. Commit and push:
    # from ~/workflow-kit
    git add -A && git commit -m "update playbooks" && git push
  3. In each project repo, pull the update:
    make sync-workflow

Architecture Rule

Keep workflow process assets in workflow-kit. Keep product code in product repos. Keep issue state in Linear. This repo is the reusable control plane — not a build plane.
